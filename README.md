# pocket_portfolio
리밸런싱을 도와주는 포트폴리오 매니징 프로그램

---
# 📊 실시간 통합 포트폴리오 트래커 (Real-time Portfolio Tracker)

**여러 증권사에 흩어져 있는 나의 모든 자산(국내외 주식, ETF, 금, 현금 등)을 한 곳에서 모아보고 실시간 수익률을 추적하는 데스크톱 애플리케이션입니다.**

---

## 🎯 제작 목적
현대 투자자들은 ISA, 연금저축, IRP, 해외직투 등 세제 혜택과 목적에 따라 여러 증권사의 다양한 계좌를 동시에 운영합니다. 이로 인해 **"내 전체 자산이 정확히 얼마인지, 총 수익률은 얼마인지, 계좌별 현황과 자산군 배분(주식/채권/금 등)은 어떻게 되어 있는지"** 한눈에 파악하기가 매우 어렵습니다.

이 프로그램은 이러한 불편함을 해결하기 위해 제작되었습니다. 클릭 한 번으로 최신 환율과 실시간 시세를 반영하여 내 자산의 현재 상태를 완벽하게 진단하고, 목표 비중에 맞춘 리밸런싱 가이드까지 제공해 줍니다.

## ✨ 주요 기능
* **🔄 실시간 시세 및 환율 연동**: `yfinance` 및 웹 크롤링을 통해 미국 주식/ETF와 한국 주식/ETF의 실시간 시세, 그리고 현재 원/달러 환율을 자동으로 가져와 원화(KRW) 기준으로 통합 계산합니다.
* **🏦 계좌별 자산 관리**: 종목별로 소속 계좌(예: 미래에셋 ISA, 삼성증권 IRP 등)를 지정하여, 각 계좌별 매수금, 평가금, 수익률을 요약해서 볼 수 있습니다. `계좌명` 입력칸은 **선택 + 직접 입력이 가능한 콤보박스**로 동작하며, JSON 불러오기 후 파일 내 `account` 값들이 자동으로 목록에 채워집니다.
* **🍰 자산군 자동 분류**: 입력된 종목명과 티커를 분석하여 **현금성 자산, 주식, 채권, 금, 원자재** 5가지 자산군으로 자동 분류하고 비중을 계산합니다.
* **📑 보고서 요약의 자산군별 성과**: 우측 텍스트 보고서 **[전체 포트폴리오 요약]** 안에서, 총 매수·총 평가 아래 **자산군별**로 **투자(매수) 금액 · 평가 금액 · 수익률(%) · 손익**을 표시합니다. 순서는 주식 → 채권 → 금 → 원자재 → 현금성 자산이며, 예수금·달러 예수금은 현금성 자산에 합산됩니다. 수익률·손익은 색상(플러스/마이너스)으로 구분됩니다.
* **⚖️ 목표 비중 기반 리밸런싱 가이드**: 사용자가 설정한 목표 자산 배분 비율(%)을 바탕으로, 현재 자산 대비 **어느 자산군을 얼마만큼 더 사고 팔아야 하는지(매수/매도 금액)** 정확하게 계산해 줍니다.
* **📈 시각화 대시보드**:
  * 개별 종목 비중 원형 차트 (Pie Chart) — **마우스 호버** 시 종목명·금액·비중(%) 툴팁 표시. 비중 5% 미만 조각은 라벨을 숨겨 겹침을 줄입니다.
  * 안전/위험 **자산군** 배분 원형 차트 (현금성 자산 포함)
  * **투자자산 비중** 원형 차트 — 현금성 자산(예수금, SGOV, CD·머니마켓 ETF 등)을 **제외한** 주식·채권·금·원자재만으로 비중 표시, 호버로 상세 확인
  * **투자원금 대비 수익률** 추이 선형 차트: 상단은 일별·주별·월별 **원금 대비 수익률(%)**, 하단은 투자 원금 vs 평가금(원)을 **별도 Y축**으로 표시해 시각적 혼동을 줄입니다. 기본은 **일간** 모드이며, **일간은 최근 20개 기록 단위 가로 스크롤**로 탐색할 수 있습니다. **월간**은 월말(해당 월 마지막 기록) 기준으로 집계합니다.
  * **📊 자산분배 추이(표)**: 보고서 창에서 **[📊 자산분배 추이]**로 열리는 메뉴에서 **일별·주별·월별·분기별**을 고르면 각각 **새 팝업**에 표 형태로 `portfolio_history`에 저장된 **자산군별 비중%와 금액**을 볼 수 있습니다. 한 칸에 `24.20% (19,836,514원)`처럼 **비중 뒤 괄호 안에 금액(원)**이 함께 표시됩니다. 집계는 좌측 **일간/주간/월간 수익률** 차트와 같이 **영업일 기준**이며, 주간은 **금요일 말** 기준, 월·분기는 **해당 기간 마지막 기록** 기준입니다. 히스토리에 `allocation` 항목이 없는 (구버전) 스냅샷만 있으면 안내 메시지가 뜰 수 있습니다.
* **🔄 보고서 수동 갱신**: 보고서 창에서 **[🔄 보고서 갱신]** 버튼을 눌렀을 때만 시세/환율을 다시 조회해 차트와 텍스트 보고서를 업데이트합니다. (기존 30초 자동 갱신은 제거)
* **🪟 투명도**: 메인 창·보고서·각 팝업 창 기본 투명도 **20%**. 하단 **투명도** 버튼으로 단계 조절 가능.
* **💾 편리한 데이터 관리**: 입력한 포트폴리오와 리밸런싱 설정값을 `.json` 파일로 저장하고 언제든 다시 불러올 수 있습니다.
* **📁 수익률 히스토리 JSON 위치 지정**: 일별 스냅샷이 쌓이는 `portfolio_history` JSON을 **기본 사용자 데이터 경로** 대신 **원하는 파일**로 지정할 수 있습니다. 메인 화면의 **포트폴리오 히스토리** 카드에서 **[파일 선택…]**으로 지정하며, 선택 내용은 설정 파일에 저장되어 다음 실행에도 유지됩니다.

## 🚀 설치 및 실행 방법

### 방법 1. 실행 파일(.exe / .app)로 바로 실행하기
패키징된 실행 파일을 바탕화면 등에 두고 **더블클릭**하여 실행합니다. (파이썬이 설치되어 있지 않아도 동작합니다.)

### 방법 2. Python 환경에서 실행하기
1. Python 3.8 이상 버전을 설치합니다.
2. 터미널에서 필요한 라이브러리를 설치합니다.

```bash
pip install yfinance pandas numpy matplotlib requests beautifulsoup4 pillow
```

> 선택 의존성: `holidays` 패키지가 있으면 한국 공휴일(음력/대체공휴일 포함) 처리가 더 정확해집니다.

```bash
pip install holidays
```

3. 실행 엔트리포인트인 `run_app.py`를 실행합니다.

```bash
python run_app.py
```

### 📦 (참고) 직접 패키징(빌드)하는 방법
코드를 수정한 뒤 나만의 독립된 실행 파일로 만들고 싶다면 아래 명령어를 사용하세요.

`run_app.py`는 실행 전에 패키징 환경(PyInstaller `--noconsole`, Pillow/Tk 브릿지 등)에서 자주 만나는 런타임 이슈를 자동 완화하는 부트스트랩 패치를 적용한 뒤 GUI를 실행합니다.

```bash
pip install pyinstaller
pyinstaller --noconsole --onefile run_app.py
```

## 📖 프로그램 사용법

1. **포트폴리오 입력 (또는 불러오기)**
   * **수동 입력/추가매수:** UI 좌측 상단의 입력폼에 `계좌명`, `종목코드/티커`(해외는 티커, 국내는 6자리 숫자코드), `종목명`, `수량`, `평단가`를 입력하고 **[주식/ETF 추가하기]** 버튼을 누릅니다.
     - `계좌명`은 **선택 + 직접 입력**이 가능한 콤보박스입니다.
     - `종목코드/티커`는 **선택한 계좌에 포함된 보유 종목만** 콤보박스 후보로 뜹니다. (표시: `티커 | 종목명`)
     - 보유 종목을 콤보박스로 선택하면 `종목명`, `보유 수량`, `평균단가`, (해외면) `매입 환율(KRW/USD)`까지 **기존 값이 자동으로 채워집니다.**
     - **추가매수 모드(기본 ON)**:
       - 이미 보유 중인 (계좌+티커)라면, 입력한 `수량/평단가`를 **(추가 매수수량, 매수가)** 로 보고
         프로그램이 **총 수량/새 평단을 자동 계산**해 보유 내역을 갱신합니다.
       - 해외 종목은 `매입 환율(KRW/USD)` 입력값이 있으면 그 값을 사용해 원가를 더 정확히 반영합니다. (국내 종목에서는 환율 입력칸이 비활성화됩니다.)
     - 현금(원화/달러)은 같은 화면의 예수금 입력칸에 넣고 **[현금 자산 추가하기]**를 누르면, **현재 계좌명 칸에 적힌 계좌**로 누적 반영됩니다.
   * **일별 수익률 히스토리 파일(선택):** 예수금 블록 **바로 아래**의 **「포트폴리오 히스토리」** 카드에서 **[파일 선택…]**으로 스냅샷을 저장·불러올 JSON 파일을 지정합니다. 최초에는 OS 기본 경로의 `portfolio_history.json`을 쓰며, 다른 PC에서 복사한 파일이나 실행 파일과 같은 폴더에 둔 JSON을 쓰려면 여기서 해당 파일을 고르면 됩니다. (**열기** 대화상자이므로 기존 파일을 고를 때 덮어쓰기 확인이 뜨지 않습니다.)
   * **파일 불러오기:** 미리 작성된 JSON 파일이 있다면 **[📂 JSON 불러오기]** 버튼을 눌러 한 번에 전체 리스트를 로드합니다.  
     불러온 뒤에는 JSON의 `portfolio.account`, `account_cash.account` 값들이 계좌명 콤보박스 후보로 자동 등록됩니다.
   * **파일 저장:**
     - **[💾 저장(덮어쓰기)]**: 마지막으로 불러온/저장한 JSON 파일에 **그대로 덮어쓰기 저장**합니다.
     - **[💾 JSON 내보내기]**: 파일명을 지정해 **다른 이름으로 저장(Save As)** 합니다. (내보낸 경로는 이후 덮어쓰기 저장의 기준 경로로 사용됩니다.)
   * **초기화:** **[초기화]** 버튼을 누르면 리스트뿐 아니라 **입력칸(계좌/티커/종목명/수량/평단/예수금/환율)** 도 함께 비워져 UI가 리프래시됩니다.

2. **목표 리밸런싱 비율 설정 (선택)**
   * 메인 화면 중앙의 **'리밸런싱 가이드 ON/OFF'** 체크박스를 켭니다.
   * 주식, 채권, 금, 원자재, 현금에 대해 본인이 원하는 목표 비중을 숫자로 입력합니다. (예: 주식 60, 채권 30, 현금 10)

3. **계산 및 분석**
   * 하단의 **[📊 계산 및 차트/수익률 보기]** 버튼을 클릭합니다.
   * 프로그램이 인터넷에서 실시간 시세와 환율을 수집합니다. (종목 수에 따라 몇 초 정도 소요될 수 있습니다.)

4. **결과 확인**
   * 계산이 완료되면 새로운 **보고서 창**이 뜹니다.
   * 우측에는 **[개별 종목 수익률]**(종목·예수금 단위 상세) 아래, **[전체 포트폴리오 요약]**이 이어집니다. 요약에는 **총 매수 금액**, **총 평가 금액**, **자산군별** 블록(주식·채권·금·원자재·현금성 자산별 투자·평가·수익률·손익), 그리고 **통합 손익금·통합 수익률**이 표시됩니다. (리밸런싱 ON이면 그 아래에 자산군 리밸런싱 가이드가 추가됩니다.)
   * 좌측 상단의 버튼들을 클릭하여 시각화된 데이터를 확인하세요:
     * **[🍩 종목별 비중]** : 개별 종목 원형 차트. 조각 위에 마우스를 올리면 **종목명·평가금액·비중**이 표시됩니다.
     * **[🍰 자산군 비중]** : 주식, 채권, 금, 원자재, **현금성 자산** 등 전체 자산군 배분. 호버로 상세 확인.
     * **[📈 투자자산 비중]** : **현금성 자산을 뺀** 투자 자산만으로 자산군 비중을 표시. 호버로 상세 확인.
     * **[🏦 계좌별 수익률]** : 계좌별 매수·평가·수익률 요약.
     * **[⚖ 리밸런싱 가이드]** : 리밸런싱 ON 상태에서 계산한 뒤, 목표 비율에 맞추기 위한 자산군별 **매수/매도 금액** 안내.
     * **[📊 자산분배 추이]** : 상단 버튼 줄 **아래**에 있는 이 버튼을 누르면 **일별·주별·월별·분기별 자산군**을 고를 수 있고, 항목마다 **새 창**에 **표**로 열립니다. 각 셀은 `비중% (평가금액)` 형식이며, 집계 방식은 아래 **일간/주간/월간 수익률** 차트와 동일한 규칙(영업일, 주=금요일 말, 월·분기=기간말 기록)을 따릅니다. **[계산 및 차트/수익률 보기]**로 쌓인 `portfolio_history`의 `allocation` 데이터가 있어야 합니다.
  * 좌측 **일간 수익률 / 주간 수익률 / 월간 수익률** 버튼으로 히스토리 차트 모드를 바꿉니다. 일간은 차트 위 **가로 스크롤바**로 최근 20개 기록 윈도우를 좌우로 이동해 확인할 수 있습니다. 각 모드는 **당일/해당 주/해당 월 말 기준 투자원금 대비 수익률**을 사용합니다(전일 대비 변화율이 아님).
  * 보고서 값(우측 텍스트, 좌측 추이 차트)을 최신 시세로 다시 반영하려면 **[🔄 보고서 갱신]** 버튼을 직접 눌러 주세요. 자동 주기 갱신은 동작하지 않습니다.

## 📜 수익률 히스토리 저장 위치
**[계산 및 차트/수익률 보기]**를 실행할 때마다 오늘 날짜 키로 일별 스냅샷(수익률, 원금, 평가금, 자산군별 금액·비중)이 **현재 지정된 히스토리 JSON 파일**에 누적 저장됩니다. 저장된 `allocation`(자산군별 `amount`·`percent`)은 보고서 창의 **[📊 자산분배 추이]**에서 일·주·월·분기 단위 **표**로 열람할 수 있습니다(위 **주요 기능 · 시각화** 참고).

### 기본 파일 경로 (경로를 바꾸지 않았을 때)
앱이 처음 실행되거나, 사용자가 **다른 히스토리 파일을 지정하지 않은** 경우 아래 경로의 `portfolio_history.json`을 사용합니다.

* **Linux:** `~/.local/share/portfolio-tracker/portfolio_history.json`  
  (전체 경로 예: `/home/사용자이름/.local/share/portfolio-tracker/portfolio_history.json`)
* **macOS:** `~/Library/Application Support/portfolio-tracker/portfolio_history.json`
* **Windows:** `%APPDATA%\portfolio-tracker\portfolio_history.json`

### 사용자 지정 히스토리 파일 (메인 화면)
* 메인 화면에서 **예수금·[현금 자산 추가하기]** 영역 **아래**에 있는 **「포트폴리오 히스토리 (일별 스냅샷) 파일」** 카드에서 **[파일 선택…]**을 누릅니다.
* **열기** 대화상자로 기존 JSON을 고릅니다. 실행 파일 옆·다운로드 폴더 등에 복사해 둔 파일을 지정할 때 **저장(다른 이름으로 저장)** 대화상자가 아니므로, OS의 **덮어쓰기 확인**은 나오지 않습니다.
* 지정한 경로는 아래 **설정 파일**에 저장되어, 다음 실행부터도 동일한 히스토리 파일을 사용합니다.

| OS | 설정 파일 (`portfolio_history_location.json`) |
|----|-----------------------------------------------|
| Linux | `~/.local/share/portfolio-tracker/portfolio_history_location.json` |
| macOS | `~/Library/Application Support/portfolio-tracker/portfolio_history_location.json` |
| Windows | `%APPDATA%\portfolio-tracker\portfolio_history_location.json` |

* 아직 히스토리 파일이 없는 위치만 쓰고 싶다면, 그 폴더에 내용이 `{}`인 빈 `.json` 파일을 만든 뒤 **[파일 선택…]**에서 그 파일을 고릅니다. 이후 **[계산 및 차트/수익률 보기]**가 성공하면 해당 파일에 스냅샷이 쌓입니다.

### 구버전·작업 폴더 호환
워크스페이스(또는 프로그램 실행 디렉터리)에 있는 `portfolio_history.json`은 **구버전 호환용**으로만, **기본 저장 경로**에 아직 파일이 없을 때 **최초 1회** 자동 마이그레이션에 사용될 수 있습니다. 사용자가 **다른 경로의 히스토리 파일**을 지정한 경우에는 이 마이그레이션을 하지 않습니다.

(리눅스의 `~/.local` 등은 숨김일 수 있어 파일 탐색기에서 **숨김 파일 표시**가 필요할 수 있습니다.)

## 🧾 실현손익 리포트 사용법 (매도 기록 + 기간별 요약)
이 기능은 **“평가손익(미실현)”**이 아니라, **직접 입력한 “매도 체결(실현손익)”**을 누적 관리하고 **일/주/월/연 단위로 합산 리포트**를 보여줍니다.

### 어디서 여나요?
- 메인 화면에서 **[🧾 실현손익 리포트]** 버튼을 누르면 별도 창이 열립니다.

### 매도 기록 입력 순서
1. **계좌 선택**
   - 상단 `계좌` 콤보박스에서 매도한 계좌를 선택합니다.
2. **종목 선택**
   - `종목` 콤보박스는 **선택한 계좌의 현재 보유 종목**만 보여줍니다.
   - 실물/고정자산(`ticker: "FIXED"`)은 매도 기록 대상에서 제외됩니다.
3. **매도일 입력**
   - 기본값은 오늘 날짜이며, `YYYY-MM-DD` 또는 `YYYY/MM/DD` 형식을 허용합니다.
4. **매도수량 / 매도가 입력**
   - `매도수량`은 **현재 보유 수량을 초과할 수 없습니다.**
   - `매도가`는 국내는 **원(₩)**, 해외는 **달러($)** 기준입니다.
5. **매도환율(해외) 입력 (해외 종목만)**
   - 해외 종목은 `매도환율(해외)`이 **필수**입니다. (KRW 기준 손익 계산을 위해 사용)
   - 기본값은 실행 시점의 환율을 자동 채움(실패 시 1400).
6. **수수료(원) / 메모 (선택)**
   - `수수료(원)`은 **항상 원화(KRW)** 기준으로 입력합니다. (해외주식이어도 원화 수수료 합산)
7. **[➕ 매도 기록 추가]** 클릭
   - 매도 기록이 저장되며, 동시에 해당 종목의 보유수량이 매도수량만큼 차감됩니다.
   - 차감 후 수량이 0이 되면 해당 보유 종목은 포트폴리오에서 제거됩니다.

### 리포트(요약/목록) 보는 법
- **[🔄 리포트 새로고침]**: 현재 저장된 매도 기록을 다시 읽어 요약/목록을 갱신합니다.
- 리포트는 다음 2개 섹션으로 구성됩니다.
  - **기간별 요약(합계)**: `일간 / 주간 / 월간 / 연간`
    - 각 줄에 다음 합계를 보여줍니다.
      - `손익`: \(매도대금 - 원가 - 수수료\)
      - `매도`: 매도대금 합계(KRW)
      - `원가`: 매도된 수량의 원가 합계(KRW)
      - `수수료`: 수수료 합계(KRW)
    - `주간` 키는 ISO week 형식으로 `YYYY-W##`로 표시됩니다.
  - **매도 기록(원문 리스트)**: 날짜 오름차순으로 매도 기록이 나열됩니다.

### 해외 종목(KRW 환산) 계산 규칙
- **매도대금(KRW)**: \(매도가(USD) \times 수량 \times 매도환율(KRW/USD)\)
- **원가(KRW)**:
  - 보유 내역에 `buy_exchange_rate`가 있으면 이를 사용합니다.
  - `buy_exchange_rate`가 없으면 **매도환율을 원가 환산에도 임시로 사용**합니다. (정확한 원가를 원하면 `buy_exchange_rate`가 포함된 포트폴리오 데이터를 유지하세요.)

### 데이터 저장/백업/가져오기 동작
- **누적 저장**: 매도 기록은 `realized_pnl_records.json`에 **계속 누적**됩니다.
- **JSON 내보내기(저장)**: 포트폴리오를 JSON으로 저장할 때, 매도 기록도 `sell_records`로 함께 포함됩니다(백업 목적).
- **JSON 불러오기(병합)**: JSON에 `sell_records`가 있으면, 기존 누적 데이터와 **병합 + 중복 제거**되어 반영됩니다.
  - 중복 제거는 `매도일/계좌/티커/수량/매도가/환율/수수료/메모` 등 조합으로 수행됩니다.

## 📂 포트폴리오 데이터 포맷 (JSON)
수동 입력이 귀찮다면 아래와 같은 구조의 `my_portfolio.json` 파일을 메모장 등으로 만들어 프로그램에서 불러올 수 있습니다.

```json
{
    "account_cash": [
        {
            "account": "미래에셋 ISA",
            "cash_krw": 32000000.0,
            "cash_usd": 0.0
        },
        {
            "account": "나무증권 직투계좌1",
            "cash_krw": 0.0,
            "cash_usd": 1008.28
        }
    ],
    "rebalance_enabled": true,
    "target_ratios": {
        "주식": 60.0,
        "채권": 30.0,
        "금": 5.0,
        "원자재": 0.0,
        "현금성 자산": 5.0
    },
    "portfolio": [
        {
            "account": "미래에셋 ISA",
            "ticker": "360750",
            "name": "TIGER 미국S&P500",
            "qty": 24.0,
            "avg_price": 21352.0
        },
        {
            "account": "미래에셋 금현물 계좌",
            "ticker": "FIXED",
            "name": "금 99.99_1kg",
            "qty": 13,
            "avg_price": 194689.0,
            "price_source": "naver_marketindex",
            "naver_marketindex": {
                "category": "metals",
                "reutersCode": "M04020000",
                "unit_price": "KRW_PER_G"
            },
            "unit_weight_g": 1
        }
    ]
}
```

### `account_cash` 필드 설명
* `account`: 예수금이 속한 계좌명
* `cash_krw`: 해당 계좌의 원화 예수금
* `cash_usd`: 해당 계좌의 달러 예수금

> 구버전 호환: 기존 JSON의 최상위 `cash_krw`, `cash_usd`만 있어도 불러오기는 가능하며, 내부적으로 레거시 통합 예수금으로 변환됩니다.

### `portfolio` 내 `ticker: "FIXED"` (실물/고정자산) 포맷
`FIXED`는 `prin`, `val`을 JSON에 수동 입력하는 대신 `qty`, `avg_price`만으로 원금/수익률을 자동 계산할 수 있습니다.

* `qty`: 보유 수량(정수/실수 가능)
* `avg_price`: **1개당 매입단가(원)** → 프로그램이 `prin = qty * avg_price`로 자동 계산
* `price_source`: 실시간 평가액 연동 소스. 현재 지원: `"naver_marketindex"`
* `naver_marketindex`:
  * `category`: 예) `"metals"`
  * `reutersCode`: 예) `"M04020000"` (국내 금)
  * `unit_price`: 예) `"KRW_PER_G"` (원/g)
* `unit_weight_g`: **1개당 중량(g)**. `unit_price`가 `KRW_PER_G`일 때 `val = (원/g) * unit_weight_g * qty`로 평가
  * 기본값은 1g로 보는 것이 안전합니다. 1kg 단위라면 `1000`으로 설정하세요.

## ⚠️ 주의사항 및 트러블슈팅
* **계산 중 응답 없음:** 인터넷 환경에 따라 해외/국내 시세를 불러오는 동안 일시적으로 창이 멈춘 것처럼 보일 수 있습니다. 보통 3~5초 내에 완료되니 잠시 기다려 주세요.
* **실행 시 빈 창만 뜨거나 오류 팝업 발생 시:** 입력한 티커(Ticker)나 한국 주식 코드가 정확한지 확인해 주세요. 네이버 금융 또는 야후 파이낸스에서 검색되는 코드를 사용해야 합니다.
* **차트 글씨 깨짐 (네모 박스 등):** 프로그램이 한글 폰트(맑은 고딕, 애플고딕 등)를 자동 감지하지만, 일부 리눅스/특수 환경에서는 폰트가 없을 수 있습니다. 시스템에 나눔고딕 등 기본 한글 폰트 설치가 필요합니다.
* **리눅스에서 Matplotlib/Tk 차트 오류:** Tk 백엔드와 Pillow `ImageTk` 조합에서 툴바·`plt.figure()` 경로가 충돌할 수 있어, 보고서 창의 그래프는 `Figure` 직접 생성 + `FigureCanvasTkAgg` 임베딩 방식을 사용합니다. 문제가 있으면 스크립트를 최신으로 유지한 뒤 재실행해 보세요.
* **팝업 창 투명도가 바로 적용되지 않을 때:** 일부 Linux 창 관리자에서는 `Toplevel` 생성 직후 투명도가 늦게 반영될 수 있어, 짧은 지연 후 한 번 더 적용하는 처리가 들어 있습니다.
* **금 평가액이 비정상적으로 큰 경우:** 네이버 금 시세는 보통 `원/g`입니다. `FIXED`에 `unit_weight_g`를 1000(=1kg)로 두면 평가액이 1000배 커집니다. 실제 보유 단위(1개가 몇 g인지)에 맞춰 `unit_weight_g`를 설정하세요.
